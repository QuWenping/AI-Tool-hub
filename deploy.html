Ollama — 本地运行开源大模型

在 Mac / Linux / Windows 上私有化部署 Llama、Qwen、Mistral 等开源模型，无需联网即可对话。

1

安装 Ollama

macOS / Linux 用户直接运行官方一键安装脚本；Windows 用户下载安装包即可。

# macOS / Linux 一键安装
curl -fsSL https://ollama.com/install.sh | sh

# Windows: 下载安装包
# https://ollama.com/download/windows
2

验证安装

安装完成后，在终端输入以下命令查看版本号，确认安装成功。

ollama --version
3

下载并运行模型

首次运行会自动下载模型，默认下载 Llama 3.2。也可以替换为 qwen2.5、mistral 等模型名。

# 运行 Llama 3.2（推荐新手）
ollama run llama3.2

# 或者运行 Qwen 2.5（中文更强）
ollama run qwen2.5

# 后台启动服务（供 API 调用）
ollama serve
4

常用命令速查

列出已下载模型、删除模型、查看运行中任务等常用操作。

# 查看本地已下载的模型
ollama list

# 删除某个模型
ollama rm llama3.2

# 查看模型信息
ollama show llama3.2

# 停止运行中的模型
ollama stop llama3.2

Claude Code — Anthropic 官方 AI 编程助手

在终端中直接调用 Claude 强大的代码能力，支持代码补全、重构、调试和全自动任务执行。

1

安装 Node.js 20+

Claude Code 依赖 Node.js 环境。如果已安装请跳过，建议版本 20 或更高。

# 检查 Node 版本（需 >= 20）
node --version

# 如未安装，使用 nvm 安装（推荐）
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.7/install.sh | bash
nvm install 20
nvm use 20
2

全局安装 Claude Code

通过 npm 一键安装到全局，安装后即可在任何项目目录中使用。

npm install -g @anthropic-ai/claude-code
3

配置 API Key 并启动

首次启动会提示输入 Anthropic API Key，从控制台获取后粘贴即可。

# 进入你的项目目录
cd ~/your-project

# 启动 Claude Code
claude

# 或者设置环境变量后启动
export ANTHROPIC_API_KEY="sk-ant-your-key"
claude
4

常用指令

进入 Claude Code 后，可以用自然语言让它帮你写代码、改 Bug、生成测试。

# Claude Code 内常用指令
/help              # 查看帮助
/compact           # 压缩对话上下文
/exit              # 退出

# 自然语言示例
"帮我重构这个函数，添加类型注解"
"给这个模块写单元测试"
"找出这段代码的内存泄漏问题"

Codex — OpenAI 云端编码 Agent

OpenAI 官方推出的命令行编码助手，支持全自动代码编写、文件操作和 Shell 命令执行。

1

安装 Node.js 20+

与 Claude Code 一样，Codex CLI 基于 Node.js 构建。

node --version
# 需返回 v20.x.x 或更高
2

安装 Codex CLI

通过 npm 全局安装 OpenAI 官方 Codex 命令行工具。

npm install -g @openai/codex
3

配置 API Key 并启动

在 OpenAI 控制台生成 API Key，设置为环境变量后启动 Codex。

# 设置 API Key（Linux/macOS）
export OPENAI_API_KEY="sk-your-openai-key"

# 设置 API Key（Windows PowerShell）
$env:OPENAI_API_KEY="sk-your-openai-key"

# 进入项目目录并启动
cd ~/your-project
codex
4

使用示例

Codex 支持全自动模式，可以直接让它完成整个功能开发。

# 让 Codex 自动完成一个功能
codex "写一个 Flask API，包含用户注册和登录接口"

# 审批模式（每一步都询问确认）
codex --approval-mode "写一个 Dockerfile"

# 查看帮助
codex --help

OpenClaw — 本地 AI 模型管理面板

为 Ollama 提供美观的 Web 管理界面，支持多模型切换、聊天记录管理和 API 调试。

1

前置条件：安装 Ollama

OpenClaw 需要 Ollama 作为后端，请先确保 Ollama 已安装并运行。

# 确认 Ollama 服务在运行
ollama --version
ollama serve
2

克隆并安装 OpenClaw

从 GitHub 克隆项目源码，进入目录后安装依赖。

# 克隆仓库
git clone https://github.com/openclaw/openclaw.git
cd openclaw

# 安装依赖（Node 20+）
npm install

# 或者使用 yarn
yarn install
3

配置并启动

复制环境变量模板，修改 Ollama 地址（默认本地即可），然后启动开发服务器。

# 复制环境变量模板
cp .env.example .env

# 编辑 .env，确认 Ollama 地址
# OLLAMA_URL=http://localhost:11434

# 启动开发服务器
npm run dev

# 构建生产版本
npm run build
npm start
4

Docker 一键部署（推荐生产环境）

如果本地已有 Docker，可以直接使用 Docker Compose 部署，无需手动安装 Node。

# 使用 Docker Compose 启动
docker-compose up -d

# 查看运行状态
docker-compose ps

# 停止服务
docker-compose down

Hermes Agent — 本地智能体框架

基于 Hermes 模型构建的自治 AI Agent 框架，支持工具调用、任务规划和多轮对话，可完全离线运行。

1

安装 Python 3.10+

Hermes Agent 基于 Python 构建，建议创建虚拟环境隔离依赖。

# 检查 Python 版本
python3 --version

# 创建虚拟环境
python3 -m venv venv

# 激活虚拟环境（Linux/macOS）
source venv/bin/activate

# 激活虚拟环境（Windows）
venv\Scripts\activate
2

克隆仓库并安装依赖

从 GitHub 获取最新源码，安装 requirements 中列出的依赖包。

# 克隆 Hermes Agent 仓库
git clone https://github.com/nousresearch/hermes-agent.git
cd hermes-agent

# 安装依赖
pip install -r requirements.txt

# 如需 GPU 加速，安装 CUDA 版 PyTorch
pip install torch torchvision torchaudio --index-url https://download.pytorch.org/whl/cu121
3

下载 Hermes 模型

通过 Hugging Face 或 Ollama 下载 Hermes 模型权重。以下示例使用 Ollama 方式，最简单。

# 通过 Ollama 下载 Hermes 模型
ollama pull hermes3

# 或者下载 Pro 版（功能调用更强）
ollama pull hermes3-pro

# 确认模型已下载
ollama list
4

配置并启动 Agent

复制配置文件模板，设置模型路径和工具 API，然后启动 Agent 服务。

# 复制配置文件
cp config.yaml.example config.yaml

# 编辑 config.yaml，设置模型
# model: "hermes3"
# ollama_url: "http://localhost:11434"

# 启动 Agent 服务
python main.py

# 或者启动 Web UI
python webui.py --port 8080
5

Docker 部署（可选）

生产环境推荐使用 Docker 部署，一次构建，到处运行。

# 构建镜像
docker build -t hermes-agent .

# 运行容器（映射 8080 端口）
docker run -d -p 8080:8080 --name hermes hermes-agent

# 查看日志
docker logs -f hermes
